# When to Use

When onboarding to a project that uses open-spec-code, or verifying the setup is complete.

# Checklist

- [ ] `open-spec-code init -u <name>` has been run
- [ ] `.osc/workflow.md` has been read and adjusted for the repo
- [ ] `.osc/spec/shared/index.md` contains at least the key convention links for this repo
- [ ] `.osc/spec/project-spec.md` has been generated (using `project-spec` skill)
